PHP 7.1 Приложение File-transfer была разработанно во время прохождения производственной практики. Задача - разработка облачного хранилища, аналога DropBox. Сервер был настроен на загрузку файлов размером до 200 мб за раз. Приложение имеет следующий функционал:

	- Регистрация новых пользователей (регистрируется в бд)
	- Авторизация пользователей (используются сессии)
	- Загрузка и хранение файлов на сервере (регистрация их в бд)
	- Удаление файлов (и из бд)
	- Скачивание файлов

При регистрации, для будущей работы с куки предусмотрено хеширование пароля с подмешением соли. При загрузке/удалении файлов в бд считается количество оставшегося места для пользователя. Во всех сценариях, необходимых для зарегистрированных пользователей, реализована проверка на наличие сессии. В случае отсутствия прав доступа (незарегистрированный пользователь), будет перенаправлен на страницу авторизации

Итак, по файлам.

- .htaccess - файл конфигурации приложения, настроен на перенаправление всех запоров на единую точку входа index.php

- data_base.php - настройка подключения к базе данных, развернутой в phpmyadmin

- delete.php - удаление данных по запросу. Удаление из бд и сервера с пересчетом оставшегося места.

- download.php - настройка страницы на скачивание файла

- index.php - настройка роутинга и обработчиков для него

- login.php - начала работы сессии и авторизация пользователей

- logout.php - завершение работы с сессией процесс получение списка файлов и базы данных и последующее отображение в приложении

- register.php - создание нового пользователя в бд и репозитории на сервере с именем пользователя (предусмотрены запрещенные символы для создание папок и предоставление прав на запись и чтение)

- session.php - инициализация сессии, проверка по существованию id пользователя

- (моя гордость(нет)) upload.php - загрузка файлов на сервер в репозитория текущего пользователя. Предусмотрены функции для обработки повторяющихся названий файлов, переход названий файлов из кириллицы на латиницу. Загрузка 1 файла ограничена 200мб.
